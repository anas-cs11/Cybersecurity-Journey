# 1. The Foundation: Asset Inventory
An Asset Inventory is a comprehensive catalog of every item that has value to the organization. This isn't just a list of laptops; it includes software, cloud storage, and even the people themselves.

Resource Allocation: Just like a shepherd needs an accurate count to provide enough food, a security team needs an inventory to know how many licenses, patches, or monitoring tools to buy.

Gap Detection: If a device that isn't on the list suddenly connects to the network, an inventory allows you to spot this "rogue" device immediately.

## 2. Asset Classification: Sorting by Value
Not all data is created equal. A security analyst must prioritize protection based on how sensitive an asset is. This is done through Asset Classification, which typically follows a four-tier structure:

| **Classification** | **Who can see it?** | **Example** |
| --- | --- | --- |
| **Public** | Anyone (External) | Marketing brochures, public website text. |
| **Internal-only** | Employees only | Employee handbook, internal memos. |
| **Confidential** | Specific teams/projects | Design drafts for a new product, project plans. |
| **Restricted** | Strictly "Need-to-Know" | Customer credit card info, health records, trade secrets. |

## 3. The Consequences of Classification
Classification labels dictate the "Lifecycle" of an asset—how it is handled, who can change it, and how it is eventually destroyed.

Disclosure: A "Restricted" file must be encrypted and never sent over public email.

Alteration: Only authorized users in the "Finance" group should have the permissions to alter a "Confidential" payroll database.

Destruction: While a "Public" flyer can be thrown in the trash, "Restricted" hard drives must be physically shredded to ensure no data can be recovered.
