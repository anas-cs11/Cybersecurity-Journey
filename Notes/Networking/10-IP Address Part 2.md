### 1. **IPv4** (Internet Protocol version 4)

- This is the **old and most commonly used** version.
- It was created in 1981.
- It uses **32-bit** addresses.
- Written in **decimal** with 4 numbers separated by dots.

**Example of IPv4 address:**

- 192.168.1.1
- 103.45.67.89
- 8.8.8.8 (Google DNS)

**Important Points about IPv4:**

- Total possible addresses = **4.3 billion** (approximately)
- Addresses look like: **xxx.xxx.xxx.xxx** (each xxx is between 0–255)
- It is still used in almost all home networks and many websites.
- **Problem**: We are running out of IPv4 addresses because there are more devices (phones, laptops, smart TVs, IoT devices) than available addresses.

### 2. **IPv6** (Internet Protocol version 6)

- This is the **newer version** designed to solve the shortage of IPv4 addresses.
- It uses **128-bit** addresses.
- Written in **hexadecimal** with 8 groups separated by colons (:).

**Example of IPv6 address:**

- 2001:0db8:85a3:0000:0000:8a2e:0370:7334
- Short form: 2001:db8:85a3::8a2e:370:7334

**Important Points about IPv6:**

- Total possible addresses = **340 undecillion** (a huge number – 340 followed by 36 zeros). Enough for many years.
- Much longer and more complicated to read than IPv4.
- Supports better security and auto-configuration.
- Many modern devices and networks already support IPv6.
- Gradually replacing IPv4, but both are used together right now (called **dual stack**).
