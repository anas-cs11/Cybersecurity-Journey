## Operating System (OS) Hardening

An **Operating System** (**OS**) is the software that acts as the intermediary between computer hardware and the user. Since the **OS** is the first program loaded when a computer turns on, securing it is essential—one insecure **OS** can lead to an entire network being compromised.

---

## 1. Regular Hardening Tasks

Some procedures must be performed at regular intervals to keep the system defended against new threats.

- **Patch Updates:** These are software and **OS** updates that fix security vulnerabilities. As soon as a vendor releases a patch, malicious actors know exactly where the weakness is in out-of-date systems.
- **Hardware and Software Disposal:** Old hardware must be wiped before being thrown away. Unused software should be deleted because popular programming languages often have known vulnerabilities that hackers can exploit.
- **Backups:** Keeping regular copies of data ensures it can be recovered if the system is compromised.

---

## 2. Baseline Configuration

A **Baseline Configuration** (also called a **Baseline Image**) is a documented set of specifications used as the standard for all future builds and updates.

- **The Goal:** It provides a "known good" state to compare against.
- **Example:** A baseline might include specific firewall rules or a list of allowed ports. If a security team suspects a hack, they compare the current settings to the **Baseline** to see what changed.

## 3. Identity and Access Policies

Hardening the way users log in is one of the most effective ways to stop unauthorized access.

| **Policy** | **Description** | **Example** |
| --- | --- | --- |
| **Strong Password Policy** | Rules that make passwords harder to guess or "crack." | Min. 8 characters, 1 capital letter, 1 number, and 1 symbol. |
| **Account Lockout** | Disabling access after too many failed attempts. | Losing network access after 5 wrong password entries. |
| **MFA** | **Multi-Factor Authentication**; requiring 2 or more ways to verify identity. | A password (something you know) + a fingerprint (something you are). |

**The Three Factors of MFA:**

1. **Something you know:** (Password or PIN).
2. **Something you have:** (ID card or physical token).
3. **Something you are:** (Biometrics like a fingerprint or facial scan).

---

## 4. Real-World Context: The Emergency Patch

**Case Study: The Programming Library Flaw**
Imagine a security team managing servers for a large company.

1. **The Threat:** A major vulnerability is found in a common programming library used by almost every application in the company.
2. **The Action:** The team performs an **Emergency Patch**. They must quickly update the **OS** on every single server to fix the hole before hackers can use it.
3. **The Result:** Once the patch is applied, the team updates their **Baseline Configuration** to ensure every new server they build in the future already has this fix included.
